# Dialog Components

A collection of accessible dialog components built on top of Radix UI's Dialog primitive. These components provide a modal dialog system with animations, styling, and proper accessibility features.

## Components

### Dialog

The root container component for the dialog system.

```jsx copy
<Dialog>
  <DialogTrigger />
  <DialogContent>
    {/* Dialog content */}
  </DialogContent>
</Dialog>
```

### DialogTrigger

The button that opens the dialog.

```jsx copy
<DialogTrigger className="custom-class">Open Dialog</DialogTrigger>
```

### DialogContent

The main container for dialog content. Renders in a portal and includes an overlay.

```jsx copy
<DialogContent className="custom-class">
  <DialogHeader>
    <DialogTitle>Title</DialogTitle>
    <DialogDescription>Description</DialogDescription>
  </DialogHeader>
  {/* Content */}
  <DialogFooter>
    {/* Actions */}
  </DialogFooter>
</DialogContent>
```

Props:
- `className`: Optional string for custom styles
- `children`: React nodes to render inside the dialog
- All other props are passed to the underlying Radix UI Dialog.Content

### DialogHeader

Container for dialog title and description with responsive styling.

```jsx copy
<DialogHeader>
  <DialogTitle>Title</DialogTitle>
  <DialogDescription>Description</DialogDescription>
</DialogHeader>
```

Props:
- `className`: Optional string for custom styles
- Accepts all HTML div element props

### DialogFooter

Container for dialog actions with responsive layout.

```jsx copy
<DialogFooter>
  <button>Cancel</button>
  <button>Confirm</button>
</DialogFooter>
```

Props:
- `className`: Optional string for custom styles
- Accepts all HTML div element props

### DialogTitle

The title of the dialog. Should be used within DialogHeader.

```jsx copy
<DialogTitle className="custom-class">Dialog Title</DialogTitle>
```

Props:
- `className`: Optional string for custom styles
- Inherits Radix UI Dialog.Title props

### DialogDescription

The description text for the dialog. Should be used within DialogHeader.

```jsx copy
<DialogDescription className="custom-class">
  Detailed description of the dialog's purpose
</DialogDescription>
```

Props:
- `className`: Optional string for custom styles
- Inherits Radix UI Dialog.Description props

## Styling

The components use Tailwind CSS for styling with the following features:

### Animations
- Fade in/out animations for overlay and content
- Zoom animations for dialog content
- Slide animations for positioning

### Responsive Design
- Full-width on mobile, max-width on larger screens
- Centered positioning
- Responsive text alignment
- Responsive footer button layout

### Theme Integration
- Uses background and text colors from theme
- Configurable through Tailwind classes
- Supports dark/light modes through CSS variables

## Technical Details

### Key Features

1. **Accessibility**
   - Built on Radix UI primitives
   - Proper ARIA attributes
   - Keyboard navigation support
   - Screen reader friendly

2. **Portal Rendering**
   - Content renders in a portal
   - Prevents z-index and stacking context issues

3. **Animation States**
   - Open/closed state animations
   - Smooth transitions
   - Hardware-accelerated transforms

## Examples

### Basic Dialog

```jsx copy
<Dialog>
  <DialogTrigger>Open</DialogTrigger>
  <DialogContent>
    <DialogHeader>
      <DialogTitle>Example Dialog</DialogTitle>
      <DialogDescription>
        This is a basic example of the dialog component.
      </DialogDescription>
    </DialogHeader>
    <p>Main content goes here.</p>
    <DialogFooter>
      <button>Action</button>
    </DialogFooter>
  </DialogContent>
</Dialog>
```

### Custom Styled Dialog

```jsx copy
<Dialog>
  <DialogContent className="max-w-xl bg-custom">
    <DialogHeader className="border-b pb-4">
      <DialogTitle className="text-xl">Custom Dialog</DialogTitle>
    </DialogHeader>
    {/* Custom content */}
  </DialogContent>
</Dialog>
```

## Accessibility

The dialog components follow WAI-ARIA guidelines for modal dialogs:

- Proper role="dialog" attribute
- Focus management
- Escape key support
- Click-outside handling
- Screen reader announcements
- Focus trap within dialog

## Browser Support

- Modern browsers supporting:
  - CSS Grid
  - CSS Custom Properties
  - CSS Transforms
  - IntersectionObserver (for animations)
  - DOM Portals
