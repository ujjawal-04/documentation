# Checkbox

The `Checkbox ` component is a wrapper around the Medusa UI Checkbox component that adds a properly associated label. It provides a consistent interface for checkbox interactions with proper accessibility attributes.

## Features

- Consistent checkbox styling
- Properly associated label
- Customizable label text
- Controlled checkbox state
- Accessibility attributes
- Test ID support

## Props

| Prop | Type | Required | Description |
|------|------|----------|-------------|
| `checked` | `boolean` | No | The checked state of the checkbox (defaults to true) |
| `onChange` | `function` | No | Function to call when the checkbox state changes |
| `label` | `string` | Yes | The text to display next to the checkbox |
| `name` | `string` | No | The name attribute for the checkbox (useful for forms) |
| `data-testid` | `string` | No | Test ID for the component |

## Usage

```tsx copy
import Checkbox from '@/components/Checkbox'
import { useState } from 'react'

export default function TermsPage() {
  const [termsAccepted, setTermsAccepted] = useState(false)
  
  return (
    <div className="my-4">
      <Checkbox
        label="I accept the terms and conditions"
        checked={termsAccepted}
        onChange={() => setTermsAccepted(!termsAccepted)}
        name="terms"
        data-testid="accept-terms-checkbox"
      />
      
      <button 
        disabled={!termsAccepted}
        className="mt-4 px-4 py-2 bg-black text-white rounded disabled:opacity-50"
      >
        Continue
      </button>
    </div>
  )
}
```

## Styling

The component uses Tailwind CSS for styling and includes:

- Consistent spacing between checkbox and label
- Flexbox layout for alignment
- Custom text styling for the label
- Proper vertical alignment with `items-start`

## States

### Default State
- Checkbox appears checked (default value is `true`)
- Label is displayed next to the checkbox

### Unchecked State
- Checkbox appears unchecked when `checked={false}` is provided
- No visual change to the label

### Interactive State
- Checkbox state toggles when clicked
- The `onChange` callback is triggered

## Accessibility

The component includes several accessibility features:

- Proper `htmlFor` association between label and checkbox
- `role="checkbox"` attribute
- `aria-checked` attribute that matches the checked state
- Clickable label that toggles the checkbox
- Support for `data-testid` for testing

## Dependencies

- @medusajs/ui (for Checkbox and Label components)
- react

## Technical Details

The component:

- Uses the Medusa UI components as a foundation
- Sets default checked state to true
- Uses a button type for the checkbox rather than a native checkbox input
- Applies custom styling to ensure consistent appearance
- Uses `!transform-none` and `!txt-medium` utility classes to override default label styling
